# git-changelog

Анализ последних коммитов фреймворка caustic с классификацией по сложности,
модулю, влиянию и статусу. Группировка связанных коммитов в цепочки.

## Аргументы

- `N` — количество коммитов (по умолчанию 30)

## Шаг 1 — Сбор данных

Выполнить:
```bash
git log --oneline -N --format="%h %ad %s" --date=short
```

## Шаг 2 — Классификация каждого коммита

Определить категорию **сложности** по сообщению коммита.
При неоднозначности — заглянуть в `git show --stat <hash>`.

| Категория | Признаки в сообщении / diff |
|-----------|----------------------------|
| Косметическое | форматирование, комменты, переименование |
| Документация | CLAUDE.md, README, только .md файлы |
| Баг-фикс | фикс, fix, исправление |
| Простая логика | упрощение, < 50 строк diff, 1-2 файла |
| Рефакторинг (средний) | вынос, рефакторинг, переименование API |
| Крупный рефакторинг | разбивка файла, выделение модуля, > 200 строк |
| Новый модуль/API | новый koh_*.h/.c, новая подсистема |
| Новая сложная логика | новая система, > 300 строк нового кода |
| Инфраструктура | сборка, bld.lua/bld.tl, koh, deploy, LSP, GDB |
| Кросс-платформа | wasm, win, mingw, emscripten, modules_* |
| Исследование | llm, исследование, эксперимент |

Определить **модуль** — одно слово:

ECS, Physics, Camera, Stage, Render, Input, Resource, Lua, Timer,
Build, Docs, Test

Определить **влияние** — кто "заметит":

| Значение | Смысл |
|----------|-------|
| потребитель | видимо в проектах-потребителях (публичный API, поведение) |
| код | только внутренняя структура фреймворка |
| сборка | инфра, CLI koh, кросс-компиляция, deploy |
| тесты | selftest, тестовые проекты |

Определить **статус**:

| Значение | Смысл |
|----------|-------|
| готово | завершённая работа |
| WIP | есть незакоммиченные продолжения или TODO |
| заброшено | начато, но не развивается |

## Шаг 3 — Таблица

Вывести таблицу:

| Дата | Коммит | Описание | Модуль | Сложность | Влияние | Статус |
|------|--------|----------|--------|-----------|---------|--------|

Жирным выделить коммиты категорий "Новый модуль/API",
"Новая сложная логика", "Крупный рефакторинг".

## Шаг 4 — Связанные коммиты (цепочки)

Сгруппировать коммиты одной задачи в цепочки. Определять по:
- Общему ключевому слову (cimgui, modules, shallow-fetch...)
- Близости по дате
- Затронутым файлам (git show --stat)

Формат вывода:
```
cimgui:    f3d894b -> 219f88c -> 00f50ac  (3, 02.10)
shallow:   19b2328  (1, 02.10)
verify:    26e81f7  (1, 02.10)
```

## Шаг 5 — Сводка по категориям

Сгруппировать по категориям сложности:
- Количество коммитов в каждой
- Перечень ключевых для "Новая сложная логика" и "Новый модуль/API"

## Шаг 6 — Период и активность

Вывести: диапазон дат, общее число коммитов, среднее в день.

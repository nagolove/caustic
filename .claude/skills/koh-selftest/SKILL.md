# Самотестирование проекта (koh-selftest)

Скилл читает список тестовых каталогов из `~/caustic/selftest.lua`,
для каждого собирает проект, проверяет пересборку бинарника
и запускает тест. В конце — сводная таблица.

## Шаг 1 — Чтение списка тестов

1. Прочитай файл `~/caustic/selftest.lua`
2. Извлеки все строки-пути (в кавычках), НЕ закомментированные
3. Закомментированные строки (с `--`) — пропусти, но запомни
4. Проверь существование каждого каталога через `ls`
5. Отсутствующие каталоги отметь как MISSING

## Шаг 2 — Определение исполняемого файла

Для каждого существующего каталога:

1. Найди ELF-исполняемый файл в корне каталога:
   ```bash
   file -h <dir>/* 2>/dev/null | grep 'ELF.*executable' | cut -d: -f1
   ```
2. Исключи файлы-скрипты (например `debug` — это GDB-скрипт, а не тест)
3. Если ELF-файлов несколько — выбери тот, чьё имя совпадает
   с последним компонентом пути каталога (с заменой `-` на `_`
   или `-` на `-`)
4. Запиши: каталог → путь к исполняемому файлу

## Шаг 3 — Сборка и запуск каждого теста

Для каждого каталога (по порядку из selftest.lua):

1. Запомни mtime исполняемого файла:
   ```bash
   stat -c '%Y' <exe_path>
   ```
2. Выполни сборку:
   ```bash
   koh make
   ```
   с `workdir` = путь к каталогу теста
3. Проверь mtime после сборки:
   ```bash
   stat -c '%Y' <exe_path>
   ```
4. Сравни: если mtime изменился → EXE_CHANGED=YES, иначе NO
5. Запусти тест:
   ```bash
   <exe_path>
   ```
6. Определи результат:
   - **PASS** — все тесты успешны (строка `N of N (100%) tests successful`)
   - **FAIL** — есть ошибки/краш (assertion, SEGV, не все тесты пройдены)
   - **CRASH** — процесс упал (SEGV, ABORTING, error exit code)

## Шаг 4 — Итоговая таблица

Выведи таблицу:

| # | Каталог | EXE изменён | Результат |
|---|---------|:-----------:|-----------|
| 1 | caustic-test-b2 | YES | 7/7 PASS |
| 2 | ... | ... | ... |

Для провалов — укажи детали:
- Имя упавшего теста
- Тип ошибки (assertion, SEGV, abort)
- Номер строки если есть

В конце — краткая сводка:
- X из Y тестов PASSED
- Список FAIL-тестов с деталями
- Список MISSING-каталогов
